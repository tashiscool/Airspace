# Demo Script

This is a short script for a 3-5 minute public-interest demo video.

## Title

**Airspace: turning weather, PIREPs, NOTAMs, reservations, and route constraints into explainable guidance**

## Opening

"Airspace is a donation-only public-interest aviation safety prototype. It is not certified and not operational. The goal is to show how live-style aviation inputs could be fused into clear, traceable guidance faster."

## Scene 1: Mission Explorer

Show:

- Mission Explorer,
- active mission list,
- weather verdict badges,
- affected active mission card,
- what changed since last brief,
- source-family chips.

Say:

"The operator starts with an operations board. Instead of reading raw weather and message traffic in separate places, the board shows which active missions need attention, what changed, and exactly which sources drove the verdict."

## Scene 2: Weather And PIREPs

Show:

- Weather page,
- real-time input cards,
- route blockage count,
- PIREP workflow card,
- coordination queue,
- selected SIGMET map feature,
- PIREP/weather guidance cards.

Say:

"Weather products and PIREPs stay distinct. Coordinate-bearing SIGMET/AIRMET/CWAP-style text gets map geometry. METAR and TAF products that do not carry route geometry remain guidance artifacts instead of fake shapes."

## Scene 3: Mission Route Impact

Show:

- Mission detail,
- route impact summary,
- affected PIREPs,
- alternative route/cost comparison,
- source chips.

Say:

"The mission page shows the operational decision: affected route segments, confidence, source count, relevant PIREPs, and a prototype reroute comparison with added distance, delay, fuel, cost, avoided hazards, and residual constraints."

## Scene 4: Coordination Draft

Click Coordinate.

Show:

- prefilled message draft,
- mission,
- recommended action,
- impact summary,
- source refs.

Say:

"The system does not send operational messages by itself. It drafts a coordination note from the cited artifacts so a human operator can review and edit it."

## Scene 5: Pilot Brief

Show:

- Pilot Brief page,
- verdict,
- route impact,
- what changed,
- source artifacts,
- printable trace summary.

Say:

"The same fused decision becomes a read-only pilot handoff. The brief answers what changed, why the route is affected, which sources mattered, and what the recommendation is."

## Scene 6: Decision Trace And Replay

Show:

- Decision page,
- summary,
- trace,
- map,
- audit/replay.

Say:

"Every decision is traceable. The result includes source refs, blocking constraints, rule trace, audit envelope, and replay bundle. The point is not black-box automation; it is faster guidance with inspectable evidence."

## Closing

"This prototype is open for public review. The most useful feedback is operational: what source artifacts are missing, what guidance would be misleading, what scenarios should be added, and what would need calibration before real-world use."
